# Communication Skills

## Giao tiếp hiệu quả cho QA

### Với Developer
- Report bug **rõ ràng, có evidence**
- Tôn trọng, không blame
- Suggest solution nếu có thể
- Phân biệt bug vs feature request

### Với Product Manager
- Communicate **risk** bằng ngôn ngữ business
- Đề xuất priority dựa trên **user impact**
- Proactive về timeline và blockers

### Với Stakeholders
- Test report **ngắn gọn, dễ hiểu**
- Focus vào metrics và trends
- Highlight risks và recommendations

### Trong Daily Standup
```
Yesterday: Completed regression testing for Payment module (95% pass)
Today: Will test new Checkout feature + verify 3 bug fixes
Blocker: Staging environment is down since morning
```

::: info
Nội dung chi tiết sẽ được bổ sung sau.
:::
