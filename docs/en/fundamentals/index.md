# Fundamentals

This section covers the foundational knowledge that **every QA Engineer needs to master** — whether you do Manual or Automation testing, whether you are a fresher or an experienced professional.

## Contents

| # | Topic | Description |
|---|---|---|
| 1 | [Testing Concepts](./testing-concepts) | QA vs QC vs Testing, 7 ISTQB Principles, Verification & Validation, Error → Defect → Failure |
| 2 | [SDLC & STLC](./sdlc-stlc) | Software Development Life Cycle, models (Waterfall, V-Model, Agile), testing process |
| 3 | [Agile & Scrum for QA](./agile-scrum) | Agile Manifesto, Scrum framework, QA in a Sprint, User Story & AC, DoD/DoR |
| 4 | [Test Design Techniques](./test-design-techniques) | EP, BVA, Decision Table, State Transition — techniques for choosing effective test cases |
| 5 | [Testing Levels](./testing-levels) | Unit → Integration → System → Acceptance, Testing Pyramid, code examples |
| 6 | [Testing Types](./testing-types) | Smoke, Sanity, Regression, Retesting, Performance, Security, Compatibility |
| 7 | [SQL for QA](./sql-for-qa) | SELECT, WHERE, JOIN, INSERT — essential SQL skills for everyday QA work |

## Learning Path

```
Start here:
  Testing Concepts → SDLC & STLC → Agile & Scrum
  → Test Design Techniques → Testing Levels → Testing Types
  → SQL for QA

Once you've mastered Fundamentals → Move on to:
  Manual Testing (if you are a manual tester)
  Automation Testing (if you want to automate)
  API Testing (recommended to learn early — highly in demand in the market)
```

::: tip Advice
Even if you are in a hurry to learn Automation or a specific tool, **don't skip this section**. Fundamentals are like the foundation of a house — if the foundation is weak, no matter how much you build on top, it will all be shaky.
:::
