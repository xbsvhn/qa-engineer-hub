# Manual Testing

Manual Testing is the foundation of every QA Engineer -- even if you move into Automation, your Manual skills remain **irreplaceable**. Exploratory testing, test case design, and bug reporting are skills you will use throughout your entire career.

## Contents

| # | Topic | Description |
|---|---|---|
| 1 | [Test Plan & Strategy](./test-plan) | Creating test plans, scope, approach, risks, ready-to-use templates |
| 2 | [Test Case Design](./test-case-design) | Writing professional test cases, Login + E-commerce examples, priority |
| 3 | [Bug Reporting](./bug-reporting) | Writing effective bug reports, Severity vs Priority, Jira workflow, bug lifecycle |
| 4 | [Exploratory Testing](./exploratory-testing) | Session-based testing, SFDPOT heuristic, Tours, combining with scripted testing |
| 5 | [Test Management Tools](./test-management-tools) | Jira for QA, TestRail, Google Sheet, choosing the right tool |

## Learning Path

```
Test Case Design → Bug Reporting → Test Plan → Exploratory Testing → Tools
  (1 week)          (3 days)        (2 days)      (3 days)           (2 days)
```

::: tip Advice
Manual Testing **is not "random clicking"**. It is a systematic process that demands analytical thinking, test case design skills, and professional communication. A QA who excels at Manual Testing will transition to Automation far more easily than someone who only knows how to code but lacks a testing mindset.
:::
