# Fundamentals

Phần này bao gồm các kiến thức nền tảng mà **mọi QA Engineer cần nắm vững** — dù bạn làm Manual hay Automation, dù bạn là fresher hay đã có kinh nghiệm.

## Nội dung

| # | Chủ đề | Mô tả |
|---|---|---|
| 1 | [Testing Concepts](./testing-concepts) | QA vs QC vs Testing, 7 nguyên tắc ISTQB, Verification & Validation, Error → Defect → Failure |
| 2 | [SDLC & STLC](./sdlc-stlc) | Vòng đời phát triển phần mềm, các mô hình (Waterfall, V-Model, Agile), quy trình testing |
| 3 | [Agile & Scrum cho QA](./agile-scrum) | Agile Manifesto, Scrum framework, QA trong Sprint, User Story & AC, DoD/DoR |
| 4 | [Test Design Techniques](./test-design-techniques) | EP, BVA, Decision Table, State Transition — kỹ thuật chọn test case hiệu quả |
| 5 | [Testing Levels](./testing-levels) | Unit → Integration → System → Acceptance, Testing Pyramid, code examples |
| 6 | [Testing Types](./testing-types) | Smoke, Sanity, Regression, Retesting, Performance, Security, Compatibility |
| 7 | [SQL cho QA](./sql-for-qa) | SELECT, WHERE, JOIN, INSERT — kỹ năng SQL thiết yếu cho QA hàng ngày |

## Lộ trình học

```
Bắt đầu từ đây:
  Testing Concepts → SDLC & STLC → Agile & Scrum
  → Test Design Techniques → Testing Levels → Testing Types
  → SQL cho QA

Sau khi nắm vững Fundamentals → Chuyển sang:
  Manual Testing (nếu bạn là manual tester)
  Automation Testing (nếu bạn muốn automate)
  API Testing (recommend học sớm — rất hot trên thị trường)
```

::: tip Lời khuyên
Dù bạn đang vội học Automation hay tool cụ thể nào, **đừng bỏ qua phần này**. Fundamentals giống như nền móng của ngôi nhà — nếu yếu thì tầng trên xây bao nhiêu cũng lung lay.
:::
