# Manual Testing

Manual Testing là nền tảng của mọi QA Engineer — dù bạn chuyển sang Automation thì kỹ năng Manual vẫn **không thể thay thế**. Exploratory testing, test case design, bug reporting đều là kỹ năng dùng suốt career.

## Nội dung

| # | Chủ đề | Mô tả |
|---|---|---|
| 1 | [Test Plan & Strategy](./test-plan) | Lập kế hoạch test, scope, approach, risks, template dùng ngay |
| 2 | [Test Case Design](./test-case-design) | Viết test case chuyên nghiệp, ví dụ Login + E-commerce, priority |
| 3 | [Bug Reporting](./bug-reporting) | Viết bug report hiệu quả, Severity vs Priority, Jira workflow, bug lifecycle |
| 4 | [Exploratory Testing](./exploratory-testing) | Session-based testing, SFDPOT heuristic, Tours, kết hợp với scripted testing |
| 5 | [Test Management Tools](./test-management-tools) | Jira cho QA, TestRail, Google Sheet, chọn tool phù hợp |

## Lộ trình học

```
Test Case Design → Bug Reporting → Test Plan → Exploratory Testing → Tools
  (1 tuần)          (3 ngày)        (2 ngày)      (3 ngày)           (2 ngày)
```

::: tip Lời khuyên
Manual Testing **không phải là "click bừa"**. Đó là quy trình có hệ thống, đòi hỏi tư duy phân tích, kỹ năng thiết kế test case, và khả năng giao tiếp chuyên nghiệp. Một QA giỏi Manual sẽ dễ dàng chuyển sang Automation hơn người chỉ biết code mà thiếu tư duy testing.
:::
