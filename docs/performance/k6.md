# K6

## K6 là gì? (WHAT)

K6 là **modern load testing tool** viết bằng Go, scripts bằng **JavaScript**. Nhẹ, nhanh, và tích hợp CI/CD dễ dàng. Được Grafana Labs phát triển.

### Tại sao K6? (WHY)

| Feature | K6 | JMeter |
|---|---|---|
| **Scripting** | JavaScript (quen thuộc) | GUI-based (XML) |
| **CI/CD** | Native support | Cần thêm plugins |
| **Performance** | Rất nhẹ, low resource | Nặng (Java-based) |
| **Code review** | Scripts trong Git | JMX files khó review |
| **Learning curve** | Dễ nếu biết JS | Dễ nếu dùng GUI |

## Test đầu tiên

```javascript
// load-test.js
import http from 'k6/http';
import { check, sleep } from 'k6';

// Configuration
export const options = {
  vus: 10,            // 10 virtual users
  duration: '30s',    // chạy 30 giây
};

// Test function — mỗi VU chạy hàm này lặp lại
export default function () {
  const res = http.get('https://api.example.com/products');

  check(res, {
    'status is 200': (r) => r.status === 200,
    'response time < 500ms': (r) => r.timings.duration < 500,
    'body has products': (r) => r.json().length > 0,
  });

  sleep(1);  // Think time 1 giây (mô phỏng user thật)
}
```

```bash
# Chạy
k6 run load-test.js
```

## Load Patterns

### Constant Load

```javascript
export const options = {
  vus: 50,
  duration: '5m',
};
```

### Ramp-up / Ramp-down (Realistic)

```javascript
export const options = {
  stages: [
    { duration: '2m', target: 50 },   // Ramp up to 50 users
    { duration: '5m', target: 50 },   // Stay at 50 for 5 min
    { duration: '2m', target: 100 },  // Ramp up to 100
    { duration: '5m', target: 100 },  // Stay at 100 for 5 min
    { duration: '2m', target: 0 },    // Ramp down to 0
  ],
};
```

### Spike Test

```javascript
export const options = {
  stages: [
    { duration: '1m', target: 50 },    // Normal load
    { duration: '10s', target: 500 },   // SPIKE!
    { duration: '2m', target: 500 },    // Stay at spike
    { duration: '10s', target: 50 },    // Back to normal
    { duration: '2m', target: 50 },     // Stay at normal
  ],
};
```

## Thresholds — Pass/Fail Criteria

```javascript
export const options = {
  stages: [
    { duration: '2m', target: 100 },
    { duration: '5m', target: 100 },
    { duration: '1m', target: 0 },
  ],
  thresholds: {
    // Response time
    http_req_duration: [
      'p(90) < 1000',   // 90% requests < 1s
      'p(95) < 2000',   // 95% requests < 2s
    ],
    // Error rate
    http_req_failed: ['rate < 0.01'],   // < 1% errors
    // Custom metrics
    'http_req_duration{name:login}': ['p(95) < 1500'],
  },
};
```

```bash
# Kết quả:
✓ http_req_duration..............: avg=450ms p(90)=890ms p(95)=1200ms
  ✓ { name:login }...............: avg=380ms p(95)=980ms
✓ http_req_failed................: 0.5% ✓ < 1%

# Nếu threshold fail:
✗ http_req_duration..............: avg=3200ms p(90)=5000ms p(95)=8000ms
  ✗ p(95) < 2000
# K6 exit code = 99 (fail) → CI/CD pipeline fail
```

## Ví dụ thực tế: E-commerce Load Test

```javascript
import http from 'k6/http';
import { check, group, sleep } from 'k6';

const BASE_URL = 'https://api.example.com';

export const options = {
  stages: [
    { duration: '2m', target: 50 },
    { duration: '5m', target: 50 },
    { duration: '1m', target: 0 },
  ],
  thresholds: {
    http_req_duration: ['p(95) < 2000'],
    http_req_failed: ['rate < 0.01'],
  },
};

export default function () {
  // Scenario: User browses products and adds to cart

  group('Browse Products', () => {
    const res = http.get(`${BASE_URL}/api/products?page=1&limit=20`);
    check(res, {
      'products status 200': (r) => r.status === 200,
      'has products': (r) => r.json().data.length > 0,
    });
    sleep(2);
  });

  group('View Product Detail', () => {
    const res = http.get(`${BASE_URL}/api/products/1`);
    check(res, {
      'product detail 200': (r) => r.status === 200,
      'has product name': (r) => r.json().name !== '',
    });
    sleep(1);
  });

  group('Add to Cart', () => {
    const payload = JSON.stringify({ productId: 1, quantity: 1 });
    const headers = { 'Content-Type': 'application/json' };
    const res = http.post(`${BASE_URL}/api/cart`, payload, { headers });
    check(res, {
      'add to cart 200': (r) => r.status === 200,
    });
    sleep(1);
  });
}
```

## CI/CD Integration

### GitHub Actions

```yaml
name: Performance Tests
on:
  schedule:
    - cron: '0 2 * * 1'  # Mỗi thứ 2, 2AM

jobs:
  k6-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: grafana/k6-action@v0.3.1
        with:
          filename: tests/performance/load-test.js
        env:
          BASE_URL: https://staging.example.com

      # Upload results
      - uses: actions/upload-artifact@v4
        if: always()
        with:
          name: k6-results
          path: summary.json
```

## K6 vs JMeter — Chọn cái nào?

| Tiêu chí | K6 | JMeter |
|---|---|---|
| **Dự án mới, team JS/TS** | ⭐⭐⭐⭐⭐ | ⭐⭐ |
| **CI/CD integration** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| **GUI-based testing** | ⭐ (no GUI) | ⭐⭐⭐⭐⭐ |
| **Protocol diversity** | ⭐⭐⭐ (HTTP focus) | ⭐⭐⭐⭐⭐ |
| **Resource efficiency** | ⭐⭐⭐⭐⭐ | ⭐⭐ |
| **Existing JMeter suite** | ⭐ | ⭐⭐⭐⭐⭐ |

**Recommendation:** Dự án mới → **K6**. Dự án có sẵn JMeter → **keep JMeter**.
