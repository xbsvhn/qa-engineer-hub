# K6

## Bản chất: K6 = JMeter phiên bản "developer-friendly"

Nếu JMeter là **máy ảnh cổ điển** (GUI, nhiều nút, ai cũng dùng được), thì K6 là **máy ảnh mirrorless** (nhỏ gọn, mạnh mẽ, cần biết cách dùng).

> **K6** = tool load testing viết bằng **JavaScript**. Nhẹ, nhanh, và tích hợp CI/CD **cực kỳ dễ**. Được Grafana Labs phát triển.

### Tại sao K6?

| Tiêu chí | K6 | JMeter |
|---|---|---|
| **Scripting** | JavaScript (quen thuộc với QA automation) | GUI-based (XML) |
| **CI/CD** | Native support -- chỉ cần 1 lệnh | Cần plugin |
| **Performance** | Rất nhẹ, ít tốn RAM | Nặng (Java-based) |
| **Code review** | Scripts trong Git, đọc được | JMX files không đọc được |
| **Learning curve** | Dễ nếu biết JS | Dễ nếu thích GUI |

::: tip Aha moment
K6 scripts là JavaScript bình thường -- bạn đã học JS cho Playwright, giờ dùng CÙNG ngôn ngữ đó để viết performance tests. Không cần học tool mới!
:::

## Test đầu tiên -- Giải thích từng dòng

```javascript
// load-test.js
import http from 'k6/http';
// ^^^ Import module HTTP của k6 -- dùng để gửi requests
import { check, sleep } from 'k6';
// ^^^ check = kiểm tra response có đúng không
//     sleep = đợi (mô phỏng người dùng thật nghỉ giữa các thao tác)

// === CẤU HÌNH: bao nhiêu người, chạy bao lâu ===
export const options = {
  vus: 10,            // 10 virtual users (10 "người dùng ảo")
  duration: '30s',    // chạy trong 30 giây
};

// === HÀM TEST: mỗi virtual user chạy hàm này LẶP LẠI ===
export default function () {
  // Gửi GET request đến server
  const res = http.get('https://api.example.com/products');

  // Kiểm tra response
  check(res, {
    'status là 200': (r) => r.status === 200,
    // ^^^ Response status phải là 200 (OK)

    'response nhanh hơn 500ms': (r) => r.timings.duration < 500,
    // ^^^ Thời gian phản hồi phải < 500ms

    'có data trả về': (r) => r.json().length > 0,
    // ^^^ Body phải có data (không rỗng)
  });

  sleep(1);
  // ^^^ Đợi 1 giây trước khi lặp lại
  // (mô phỏng người dùng thật: đọc trang rồi mới click tiếp)
}
```

```bash
# Chạy test
k6 run load-test.js

# K6 sẽ hiển thị kết quả:
#   http_req_duration...: avg=234ms  p(90)=450ms  p(95)=680ms
#   http_req_failed.....: 0.00%
#   vus.................: 10
#   iterations..........: 280
```

## Load Patterns -- Mô phỏng các kiểu tải khác nhau

### Constant Load = "Giữ nguyên số khách"

```javascript
// Đơn giản: 50 users chạy 5 phút
export const options = {
  vus: 50,           // 50 người dùng ảo
  duration: '5m',    // chạy 5 phút
};
```

### Ramp-up / Ramp-down = "Tăng dần, giảm dần" (thực tế nhất)

```javascript
export const options = {
  stages: [
    // stages = các "giai đoạn" -- giống nhạc có nhiều đoạn
    { duration: '2m', target: 50 },   // 2 phút đầu: tăng dần từ 0 lên 50 users
    { duration: '5m', target: 50 },   // 5 phút tiếp: giữ 50 users (đo tải ổn định)
    { duration: '2m', target: 100 },  // 2 phút: tăng lên 100 users
    { duration: '5m', target: 100 },  // 5 phút: giữ 100 users
    { duration: '2m', target: 0 },    // 2 phút cuối: giảm dần về 0 (kết thúc)
  ],
};
```

### Spike Test = "Đột ngột tăng vọt"

```javascript
export const options = {
  stages: [
    { duration: '1m', target: 50 },    // Tải bình thường
    { duration: '10s', target: 500 },   // SPIKE! tăng từ 50 lên 500 trong 10 giây
    { duration: '2m', target: 500 },    // Giữ tải cao 2 phút
    { duration: '10s', target: 50 },    // Hạ về bình thường
    { duration: '2m', target: 50 },     // Giữ tải bình thường -- hệ thống có recover?
  ],
};
```

## Thresholds = "Điều kiện PASS/FAIL" tự động

Thresholds giống như bạn nói trước: "Nếu P95 > 2 giây hoặc error > 1% thì = FAIL". K6 sẽ **tự động** đánh giá PASS/FAIL.

```javascript
export const options = {
  stages: [
    { duration: '2m', target: 100 },
    { duration: '5m', target: 100 },
    { duration: '1m', target: 0 },
  ],

  thresholds: {
    // === ĐIỀU KIỆN PASS/FAIL ===

    http_req_duration: [
      'p(90) < 1000',   // 90% requests phải nhanh hơn 1 giây
      'p(95) < 2000',   // 95% requests phải nhanh hơn 2 giây
    ],

    http_req_failed: [
      'rate < 0.01',    // Error rate phải < 1%
    ],

    // Threshold riêng cho từng endpoint
    'http_req_duration{name:login}': [
      'p(95) < 1500',   // Login API phải nhanh hơn 1.5 giây
    ],
  },
};
```

### Kết quả khi PASS:

```
v  http_req_duration........: avg=450ms  p(90)=890ms  p(95)=1200ms
     v { name:login }.......: avg=380ms  p(95)=980ms
v  http_req_failed...........: 0.5%  v < 1%
```

### Kết quả khi FAIL:

```
x  http_req_duration........: avg=3200ms  p(90)=5000ms  p(95)=8000ms
     x p(95) < 2000
# K6 exit code = 99 (FAIL) --> CI/CD pipeline sẽ FAIL
# --> Deploy bị BLOCK tự động!
```

::: tip Aha moment
Thresholds + CI/CD = **tự động chặn deploy** khi performance không đạt. Không cần QA ngồi xem kết quả rồi quyết định -- K6 quyết định giúp bạn.
:::

## Ví dụ thực tế: E-commerce Load Test

```javascript
// ecommerce-load-test.js
import http from 'k6/http';
import { check, group, sleep } from 'k6';
// ^^^ group = nhóm các requests lại thành 1 scenario (dễ đọc report)

const BASE_URL = 'https://api.example.com';

// === CẤU HÌNH ===
export const options = {
  stages: [
    { duration: '2m', target: 50 },   // Tăng dần lên 50 users
    { duration: '5m', target: 50 },   // Giữ 50 users trong 5 phút
    { duration: '1m', target: 0 },    // Giảm về 0
  ],
  thresholds: {
    http_req_duration: ['p(95) < 2000'],  // 95% requests < 2 giây
    http_req_failed: ['rate < 0.01'],      // Error < 1%
  },
};

// === SCENARIO: Người dùng xem sản phẩm và thêm vào giỏ hàng ===
export default function () {

  // Bước 1: Xem danh sách sản phẩm
  group('Xem danh sách sản phẩm', () => {
    const res = http.get(`${BASE_URL}/api/products?page=1&limit=20`);
    check(res, {
      'danh sách trả về 200': (r) => r.status === 200,
      'có sản phẩm': (r) => r.json().data.length > 0,
    });
    sleep(2);
    // ^^^ Người dùng đọc danh sách 2 giây trước khi click vào sản phẩm
  });

  // Bước 2: Xem chi tiết 1 sản phẩm
  group('Xem chi tiết sản phẩm', () => {
    const res = http.get(`${BASE_URL}/api/products/1`);
    check(res, {
      'chi tiết trả về 200': (r) => r.status === 200,
      'có tên sản phẩm': (r) => r.json().name !== '',
    });
    sleep(1);
    // ^^^ Đọc chi tiết 1 giây
  });

  // Bước 3: Thêm vào giỏ hàng
  group('Thêm vào giỏ hàng', () => {
    const payload = JSON.stringify({
      productId: 1,
      quantity: 1,
    });
    const headers = { 'Content-Type': 'application/json' };

    const res = http.post(`${BASE_URL}/api/cart`, payload, { headers });
    check(res, {
      'thêm giỏ hàng thành công': (r) => r.status === 200,
    });
    sleep(1);
  });
}
```

## CI/CD Integration -- Chạy K6 tự động trong GitHub Actions

```yaml
# .github/workflows/performance.yml
name: Performance Tests

on:
  schedule:
    - cron: '0 2 * * 1'
    # ^^^ Mỗi thứ 2, 2AM UTC (9AM Vietnam)
    # Chạy hàng tuần để theo dõi performance theo thời gian

jobs:
  k6-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      # Dùng K6 action có sẵn -- không cần cài đặt gì
      - uses: grafana/k6-action@v0.3.1
        with:
          filename: tests/performance/load-test.js
          # ^^^ Đường dẫn đến file test K6
        env:
          BASE_URL: https://staging.example.com
          # ^^^ Chạy trên staging, KHÔNG chạy trên production!

      # Lưu kết quả để xem sau
      - uses: actions/upload-artifact@v4
        if: always()
        with:
          name: k6-results
          path: summary.json
```

::: tip Aha moment
K6 + GitHub Actions = **performance testing tự động hàng tuần**. Nếu tuần này P95 tăng từ 800ms lên 2000ms, bạn biết NGAY -- thay vì đợi user phàn nàn.
:::

## Tóm tắt chương

| Concept | Ẩn dụ | K6 cách dùng |
|---|---|---|
| **VUs** | Người dùng ảo | `options.vus` hoặc `stages` |
| **Duration** | Chạy bao lâu | `options.duration` |
| **Stages** | Các giai đoạn (tăng, giữ, giảm) | Ramp-up, sustain, ramp-down |
| **Checks** | Kiểm tra response | `check(res, { ... })` |
| **Thresholds** | Điều kiện PASS/FAIL | `options.thresholds` -- tự động fail CI/CD |
| **Groups** | Nhóm requests | `group('tên', () => { ... })` |
| **Sleep** | Người dùng nghỉ | `sleep(1)` -- mô phỏng realistic |
| **CI/CD** | Chạy tự động | GitHub Actions + grafana/k6-action |
